# تطبيق الذاكرين (AlDhakereen) - نسخة Flutter الاحترافية

تطبيق إسلامي تم تطويره باستخدام Flutter لضمان أقصى أداء وتجربة مستخدم أصلية (Native).

## المميزات الجديدة
- **نظام CMS**: لوحة تحكم مدمجة في المجلد `web/cms` تتيح إضافة وتعديل كافة الأقسام.
- **تحديث ديناميكي**: يتم جلب المحتوى من ملف `assets/data/content.json` الذي يمكن تحديثه عبر الـ CMS.
- **بناء آلي**: عند أي تغيير في المحتوى على GitHub، يتم بناء نسخة APK جديدة آلياً عبر GitHub Actions.
- **تخصيص كامل**: إمكانية تغيير الصور، الخلفيات، الشعار، ومعلومات المطور من لوحة التحكم.

## 📄 صياغة ملف JSON للاستيراد

### 1. استيراد لقسم محدد (في صفحة القسم)
استخدم التنسيق التالي عند الاستيراد داخل قسم معين (مثل الأدعية):
```json
[
  { "title": "عنوان الموضوع", "content": "نص المحتوى..." }
]
```

### 2. استيراد شامل لكافة الأقسام (Universal Batch)
استخدم التنسيق التالي في صفحة "استيراد شامل بالدفعة" لتوزيع المواضيع آلياً:
```json
[
  {
    "section": "duas",
    "title": "اسم الدعاء",
    "content": "نص الدعاء..."
  },
  {
    "section": "quran",
    "title": "اسم السورة",
    "content": "نص السورة..."
  }
]
```
**أسماء الأقسام البرمجية المتاحة:** `quran`, `duas`, `visits`, `tafsir`, `dreams`, `stories`, `imam_ali`.

### 🖼️ إدارة الوسائط والصور
- **الشعار والخلفية:** يمكن تغييرهما من قسم "الهوية والوسائط". يدعم التطبيق روابط الإنترنت المباشرة (http) أو اختيار صور من الاستوديو ليتم تحويلها وحفظها في قاعدة البيانات.
- **أيقونات الأقسام:** يمكن تغيير أيقونة أي قسم بكتابة اسم الأيقونة من مكتبة [Material Icons](https://fonts.google.com/icons).

## هيكلية المحتوى
يتم تخزين المحتوى في `assets/data/content.json` ويشمل:
- القرآن الكريم (30 جزء)
- الأدعية والزيارات
- التفسير والقصص وتفسير الأحلام
- موسوعة الإمام علي (ع)

## لوحة التحكم (CMS)
للوصول للوحة التحكم، افتح ملف `web/cms/index.html` من خلال خادم محلي (Local Server) أو رفعه على GitHub Pages. تتيح لك:
1. إضافة عناصر فرادى أو كمجموعات (Batch Add).
2. تعديل معلومات المطور وقسم "حول".
3. ضبط مسارات الصور والشعارات.
4. الحفظ المباشر في GitHub (يتطلب Personal Access Token).

## بناء التطبيق
يتم البناء آلياً عبر GitHub Actions لإنتاج ملف APK متوافق مع كافة الأجهزة الحديثة (64 بت فقط).
يتم تشغيل البناء عند تعديل ملف البيانات أو الكود المصدري.

## 🛠️ تفاصيل الربط التقني والمزامنة (Technical Architecture)

يعتمد التطبيق على هندسة بيانات متطورة تضمن تحديث المحتوى دون الحاجة لتحديث التطبيق من المتجر:

### 1. كيف يتصل التطبيق بالمستودع؟
- يستخدم التطبيق محرك مزامنة (Hybrid Data Engine) مبرمج بلغة Dart.
- عند تشغيل التطبيق، يقوم بطلب (HTTP GET) للرابط الخام (Raw URL) لملف `assets/data/content.json` من GitHub.
- **رابط المزامنة:** `https://raw.githubusercontent.com/techtouchAI/Islamic/main/assets/data/content.json`

### 2. آلية التخزين الدائم (Local Persistence)
- بمجرد جلب البيانات من GitHub، يقوم التطبيق بمقارنتها بالنسخة الحالية.
- إذا كانت البيانات أحدث، يتم حفظها فوراً في **ذاكرة الهاتف الدائمة** (مجلد المستندات الخاص بالتطبيق).
- في المرة القادمة التي يفتح فيها المستخدم التطبيق، سيقرأ من "النسخة المحملة" أولاً، مما يضمن بقاء جميع إضافاتك من الـ CMS فعالة.

### 3. دور لوحة التحكم (CMS)
- لوحة التحكم (CMS) هي "الكاتب" في هذه العملية. هي تستخدم (GitHub API) لتعديل الملف في المستودع.
- بمجرد ضغطك على "حفظ وإرسال"، يُحدث الملف على GitHub، ثم يقوم تطبيق كل مستخدم بجلب هذا التعديل تلقائياً في المرة القادمة التي يفتح فيها التطبيق.

### 4. حماية الملفات في المستودع
- تم ضبط السكربتات البرمجية لتستخدم نظام (Targeted Update)؛ أي أنها تعدل أسطر الكود المطلوبة فقط ولا تقوم بمسح أو إعادة كتابة المستودع بالكامل، مما يحمي الملفات التي تم رفعها يدوياً أو عبر الـ CMS من الحذف العشوائي.

### 5. تحديث المحتوى يدوياً
إذا قمت بتعديلات ولم تظهر فوراً، يمكنك الذهاب إلى **الإعدادات > نظام البيانات والمزامنة** والضغط على "تحديث المحتوى الآن" لإجبار التطبيق على جلب آخر نسخة من GitHub Raw وتخزينها محلياً.
